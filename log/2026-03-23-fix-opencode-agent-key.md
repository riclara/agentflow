# Fix: OpenCode Config Format Bugs

**Date:** 2026-03-23

## Description

`opencode.json` generated by agentflow had two bugs that prevented opencode from loading:

### Bug 1: Wrong key name
Error: `Config file at opencode.json is invalid: Unrecognized key: "agents"`

opencode expects `"agent"` (singular), agentflow was writing `"agents"` (plural).

### Bug 2: Wrong value format
Error: `Invalid input: expected object, received string agent.planner`

opencode expects each agent to be an object `{ "path": "..." }`, agentflow was writing plain strings.

**Before (broken):**
```json
{ "agents": { "planner": ".opencode/agents/planner.md" } }
```

**After (correct):**
```json
{ "agent": { "planner": { "path": ".opencode/agents/planner.md" } } }
```

## Root Cause

`mergeOpencodeJson()` in `src/core/merger.ts` and `renderOpenCodeAgentsConfig()` in `src/templates/opencode/config-json.ts` were both using incorrect key names and value formats. The opencode config schema was never validated during development.

## Files Changed

1. **src/core/merger.ts** — Changed key `agents` → `agent`, type `Record<string, string>` → `Record<string, { path: string }>`
2. **src/templates/opencode/config-json.ts** — Changed return type and values from strings to `{ path: "..." }` objects
3. **README.md** — Documentation reference `agents` → `agent`
4. **README.es.md** — Documentation reference `agents` → `agent`
5. **docs/agentflow-spec.md** — 4 documentation references updated, fixed broken tree indentation

## Verified

- ✅ `npm run build` — compiles cleanly
- ✅ `npm test` — 33/33 tests pass
- ✅ `npx opencode-ai debug config` — loads all 4 agents correctly with models, prompts, and permissions
